# 楽天アフィリエイト運用メモ

## 基本方針

- AIに楽天リンクのURLを自由生成させない。
- 人が取得して確認したリンクを台帳化し、AIは `id` を選ぶだけにする。
- 宿の比較記事は `ホテル個別リンク`、エリア記事やモデルコース記事は `エリアリンク` を基本にする。
- レンタカーは専用リンク経由のみ成果対象なので、宿リンクやエリアリンクで代用しない。

## 運用の全体像

1. 楽天アフィリエイト管理画面でリンクを取得する
2. `data/affiliate-links.sample.json` を元に台帳へ追加する
3. 記事ごとに `knowledge/article-affiliate-template.json` の形式で、使いたい `linkId` を決める
4. `scripts/render_affiliate_cards.py` で、記事差し込み用の `affiliate_cards` を生成する
5. 生成した結果を記事データまたは記事HTMLに貼る

## リンク台帳で持つ項目

- `id`
  - AIやスクリプトから参照する一意キー
- `service`
  - いまは `rakuten_travel`
- `entityType`
  - `hotel` / `area` / `transport` / `rentalcar` / `activity`
- `area`
  - 指宿、石垣島、沖縄本島など
- `hotelName`
  - 個別ホテルの場合のみ
- `tags`
  - 一人旅、記念日、温泉など
- `label`
  - 管理用。リンクの説明文
- `url`
  - 実際に差し込む楽天アフィリエイトリンク
- `officialUrl`
  - 公式サイトの確認用。必要なものだけ
- `notes`
  - どういう記事で使うか

## AIにさせること

- 記事テーマから、どの `linkId` が合うかを選ぶ
- 予約導線カードの `title` / `body` / `cta` を記事に合わせて調整する
- 宿紹介カードの直下に入れる個別リンクを選ぶ

## AIにさせないこと

- 楽天リンクURLの自作
- 宿名に似た別施設のリンクを推測で差し込むこと
- レンタカー用リンクを宿泊導線へ流用すること
- バナーHTMLをそのまま乱用すること

## このサイトでの使い分け

### ホテル紹介記事

- 各ホテルカードの下に個別リンクを置く
- 記事末にはエリアリンクを置いてよい

### モデルコース記事

- 記事末の予約導線はエリアリンクを中心にする
- 必要なら航空券やレンタカーを別カードで出す

### 比較記事

- ホテルごとに個別リンク
- 記事末にエリアリンク

## HTMLの形

生成スクリプトでは、楽天リンクを以下のボタンへ差し込む前提にする。

```html
<a class="button" href="楽天リンク" target="_blank" rel="nofollow sponsored noopener">楽天トラベルで見る</a>
```

## 注意

- 画像バナーを使う場合でも、本文内の主導線はテキストまたはボタンにする
- 楽天トラベル画像の扱いは公式ガイドに従う
- API画像は掲載ルールがあるので、通常の宿紹介では手元写真や自前画像を優先する
